# Phase 0 設計提案 — 「ひと夏の一日」

> この文書は実装前の設計案です。**人間の承認を得てからコードを書きます**（CLAUDE.md準拠）。
> 決定済みの方針: シグネチャ表現=「空グラデ＋流れる雲＋光」／1日=約15分／普遍的な夏のモチーフで素朴に。

---

## 1. 全体設計の方針（平易版）

ゲームは「**毎フレーム、画面を描き直し続ける**」仕組みで動きます。パラパラ漫画と同じで、1秒に約60回、今の状態を見て絵を描き直します。これを**ゲームループ**と呼びます。

3つの管理役を分けて持ちます。

- **時間の管理役（Clock）**: ゲーム内の時刻を少しずつ進める。1日=15分。今が朝/昼/夕方/夜のどのあたりかを「0.0〜1.0の数値」で持ち、空の色や音の切り替えに使う。
- **場面の管理役（SceneManager）**: 縁側・原っぱ・神社…といった一枚絵の場面を切り替える。今どの場面にいるか、隣にどの場面があるかを持つ。
- **描画の管理役（Renderer）**: 空→遠景→地面→前景→キャラ→エフェクトの順に**レイヤーで重ねて**描く。

### 後から画像に差し替えられる構造（CLAUDE.md準拠）
描画の各パーツを「部品（レイヤー）」にして、**「コードで描く版」と「画像を貼る版」を切り替える共通の窓口**を作ります。

```js
// 描画部品の共通インターフェース
{
  drawCode(ctx, t)   // いまはこちらだけ実装（図形・グラデで描く）
  drawImage(ctx, t)  // 将来 assets/ に絵を置けばこちらに切替（Gemini製の水彩画など）
}
```

最初は全部コード描画版で完成させ、`assets/scenes/engawa.png` のような画像を置いたら自動でそちらを使う、という設計にします。**絵を描き直すのではなく、差し替えるだけ**で格上げできます。

## 2. 時間進行のデータ構造案（Cradleのフェーズ駆動を踏襲）

時間帯ごとの設定を**データで持ち**、時刻に応じて切り替え・補間します。

```js
const DAY_LENGTH_SEC = 900 // 1日=15分

// 時間帯のキーフレーム（at = 0.0〜1.0 の位置）
const PHASES = [
  { key: 'morning', at: 0.00, label: '朝',   ambient: 'morning', sky: SKY.morning },
  { key: 'noon',    at: 0.30, label: '昼',   ambient: 'noon',    sky: SKY.noon    },
  { key: 'evening', at: 0.62, label: '夕方', ambient: 'evening', sky: SKY.evening },
  { key: 'night',   at: 0.85, label: '夜',   ambient: 'night',   sky: SKY.night   },
]

// 空の色は隣り合うキーフレームの間を「線形補間」でじわじわ移す（急に変えない）
// 例: t=0.45 なら noon と evening の間を 0.45→比率で混ぜる

// 起きること（時間帯限定イベント）も at の範囲で持つ
const EVENTS = [
  { at: [0.02, 0.08], key: 'radio',     scene: 'engawa', label: '朝のラジオ体操' },
  { at: [0.30, 0.55], key: 'thunderhead',scene: 'any',   label: '入道雲がわく'    },
  { at: [0.62, 0.78], key: 'sunset',    scene: 'any',     label: '夕焼け'          },
  { at: [0.88, 1.00], key: 'stars',     scene: 'any',     label: '星空（夜は花火も検討）' },
]
```

各**場面（シーン）**も、時間帯ごとの「上書き」を持てます（例: 神社は木陰なので昼でも少し暗くする）。これで「データを足すだけ」で世界が増えます。

## 4. 描画方針とアートトークン

### Canvasでどう描くか
- **空（主役）**: 上から下への縦グラデーション。時間帯のパレットを線形補間で混ぜ、じわじわ移ろわせる。
- **雲**: やわらかい白の楕円を複数、薄く重ねてぼかし、ゆっくり横へ流す。昼は入道雲をもくもくと大きく。
- **光**: 画面上方から斜めに差す半透明の光の帯。朝は淡い金、昼は白、夕方は橙。
- **地面・前景**: 図形＋グラデ＋軽いノイズで草地・縁側・石段などを様式化（リアルより雰囲気）。
- **キャラ**: 単純な形のシルエット＋淡い色。輪郭をくっきりさせすぎない。
- **マイクロ表現（軽量）**: 草の揺れ、蝉が飛び立つ、夕方の長い影、夜の星と蛍の瞬き。

### シグネチャ表現
**「時間とともに移ろう空グラデ＋ゆっくり流れる雲＋差し込む光」**。全場面で共通の空が、その日の時間の流れを静かに語る主役になります。

## 5. 環境音の調達計画

時間帯・場面でループ再生を切り替えます。最初のタップで音を立ち上げ（iOS自動再生制限対策）、音量・ミュートを用意。

| 音 | 用途（時間帯/場面） | 調達先の当て | ライセンス方針 |
|---|---|---|---|
| アブラゼミ（ジワジワ） | 昼/全般 | Freesound（CC0で絞り込み）/ OpenGameArt | CC0優先 |
| ヒグラシ（カナカナ） | 夕方 | Freesound CC0 | CC0優先 |
| ウグイス・遠い鶏 | 朝 | Freesound CC0 | CC0優先 |
| 川のせせらぎ | 川辺 | Freesound CC0 / OpenGameArt | CC0優先 |
| 風鈴 | 縁側 | Freesound CC0 | CC0優先 |
| 虫の音・カエル | 夜 | Freesound CC0 | CC0優先 |

- **方針**: ライセンスが明確な **CC0 を最優先**。出典URLとライセンスを取得時に `CREDITS.md` へ**全数記録**（CLAUDE.md準拠）。不明な素材は使わない。
- 実際の取得は音声フェーズ（P3）で行い、それまでは無音でも動く作りにします。

## 6. 場面構成の確定案（1日目）

SPEC 3章の5場面を**全部1日目に入れる**案にします（密度より「行き来する楽しさ」を優先、各場面は軽量に）。隣接でつなぎます。

```
        [神社]
          │
[原っぱ]─[縁側]─[田んぼ道]─[川辺]
  (虫)   (拠点)   (夕暮れ)  (せせらぎ)
```

- **縁側**（拠点・一日の始まりと終わり／風鈴）
- **原っぱ**（虫採りの中心）
- **神社**（木陰・石段・住人と会話）
- **田んぼ道**（移動の道・夕暮れが映える）
- **川辺**（せせらぎ・泳ぎは2日目以降でスコープ外）

> 作る順番は「縁側を完成見本（縦の薄切り）として先に仕上げ→他場面を同じ型で量産」。これで品質の基準を先に固めます。

## 7. 実装計画（フェーズ分割）と人間の手作業チェックリスト

各フェーズは小さく区切り、着手前に改善案を1〜3個提示、実装後に自己検証（ビルド＋headless表示確認＋コンソールエラーなし）してから報告します。

| Phase | 内容 | 完了の目安（/goal登録粒度） |
|---|---|---|
| **P1** | エンジン土台（ループ/時計/場面管理/差し替え窓口）＋**縁側**でシグネチャの空 | 縁側で空の色が時間帯につれ滑らかに変わる |
