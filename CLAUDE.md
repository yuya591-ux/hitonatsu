# CLAUDE.md — 「ひと夏の一日」恒久ルール

## プロジェクト概要
- 個人の趣味プロジェクト。販売・公開予定なし。完全な自己満足のために作る
- 「夏の田舎で過ごす、朝から夜までの一日」を、固定カメラの一枚絵的な世界で味わうゲーム
- ノスタルジックな1990〜2000年代の夏休みの空気の再現が最上位の目的。ゲームとしての難易度・クリア条件は重視しない
- 詳細仕様は `SPEC.md` を正とする。本ファイルは全期間で守る恒久ルール

## 技術スタック（固定）
- HTML + CSS + JavaScript（バニラ。重量級ゲームエンジンは使わない）
- 描画は Canvas API（2D）。背景・キャラ・エフェクトはすべてコードで描画する
- 音声は Web Audio API / HTML Audio。環境音はフリー素材（後述）を再生
- ビルドツールは Vite（開発・プレビュー・本番ビルド用）
- ホスティングは **GitHub Pages**（静的サイト。追加認証・課金設定が不要でリモート運用と相性が良いため確定。理由を README に記載）
- **外部APIに依存しない**。画像生成API・課金APIは組み込まない。個人APIキーへの依存をゼロにする（「自分がいなくなっても動く」原則）

## 開発プロセス
1. **Phase 0 必須**: 実装前に、設計方針・画面構成のASCIIワイヤーフレーム・時間進行のデータ構造案・描画方針（空や雲をどう描くか）・アートトークン案を提示し、人間承認後に着手する
2. 各実装フェーズの着手前に、改善案を1〜3個提示する（「小さい追加」の範囲を守る）
3. **investigate-first**: 既存コードを変更する際は、必ず先に構造・関数名を調査する
4. 実装後の**自己検証必須**: ビルド／lint／headlessブラウザでの表示確認（描画が崩れていないか、コンソールエラーが出ていないか）まで済ませてから報告する
5. 人間の手作業は認証・デプロイ承認類のみ。CLIで完結できない操作はまとめてチェックリスト形式で提示する
6. デプロイまで Claude Code が実行する

## 差し替え可能性（重要・後からの格上げ前提）
- 背景・キャラクターは「コード描画」で作るが、**後で画像ファイル（Geminiで作った水彩画風の絵など）に差し替えられる構造**にしておく
- 具体的には、描画処理を部品化し、「コードで描く版」と「画像を貼る版」を切り替えられる設計にする。最初はコード描画版で完成させる
- 音声素材も同様に、ファイル差し替えが容易な構造にする

## 著作権（絶対遵守）
- 「僕の夏休み」など既存作品の絵・音・キャラクター・地名・固有の台詞・固有のキャラ名を一切使わない・模倣しない
- 作るのは「あの時代の夏の空気」という普遍的な雰囲気を持つ、完全オリジナルの世界
- 環境音（蝉・風鈴・川・ヒグラシ等）は、ライセンスが明確なフリー素材（CC0等）のみ使用し、出典とライセンスを `CREDITS.md` に記録する

## 情報・素材の誠実性
- 使用したフリー素材の出典・ライセンスURLを必ず `CREDITS.md` に全数記録する
- ライセンス不明の素材は使わない

## 表記
- UI文言・コードコメント・ドキュメントはすべて日本語
- 画面に出す文言は素朴で温かい言葉づかい。システム用語を露出させない
