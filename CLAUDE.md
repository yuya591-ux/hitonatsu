# CLAUDE.md — 「ひと夏の一日」恒久ルール

## プロジェクト概要
- 個人の趣味プロジェクト。販売・公開予定なし。完全な自己満足のために作る
- 「夏の田舎で過ごす、朝から夜までの一日（→数日）」を、歩いて味わうゲーム
- **最上位の目的＝「あの時代の夏の空気」の再現＝ノスタルジー・癒し**。ゲームとしての難易度・クリア条件は重視しない
- **時代背景は昭和後期〜平成初期（おおむね1970年代後半〜1990年代前半）の日本の田舎の夏休み**。僕の夏休みの空気感を常に意識する（※絵・音・固有名・台詞の模倣は禁止＝下記「著作権」）。新しすぎる物（スマホ・現代車・LED看板等）は出さない。小物・建築・生活感・音をこの時代に合わせる
- 詳細仕様は `SPEC.md` を参考にしつつ、現在の主軸は3D版（下記）。本ファイルは全期間で守る恒久ルール

## 技術スタック
- HTML + CSS + JavaScript（バニラ）。ビルドは Vite、ホスティングは **GitHub Pages**（静的・追加認証/課金不要）
- **描画の主軸は 3D（Three.js）**。`proto3d.html`／`src/proto3d/main.js`。様式＝**低ポリ＋トゥーン（やわらか）**。フォトリアルは目的に反するため狙わない（色・光・空気・接地・テクスチャ・音・間で“絵”に近づける）。※Three.js は描画ライブラリであり、重量級ゲームエンジン不使用の趣旨は維持
- 旧2D版（`index.html`／Canvas 2D）は初期プロトタイプとして残置。学びは3Dへ移植
- 音声は Web Audio API / HTML Audio。環境音はライセンス明確なフリー素材（CC0等）か自前合成のみ
- **外部APIに依存しない**。画像生成API・課金APIは組み込まない。個人APIキーへの依存をゼロにする（「自分がいなくなっても動く」原則）

## 開発プロセス
1. **Phase 0 必須**: 実装前に、設計方針・画面構成のASCIIワイヤーフレーム・時間進行のデータ構造案・描画方針（空や雲をどう描くか）・アートトークン案を提示し、人間承認後に着手する
2. 各実装フェーズの着手前に、改善案を1〜3個提示する（「小さい追加」の範囲を守る）
3. **investigate-first**: 既存コードを変更する際は、必ず先に構造・関数名を調査する
4. 実装後の**自己検証必須**: ビルド／lint／headlessブラウザでの表示確認（描画が崩れていないか、コンソールエラーが出ていないか）まで済ませてから報告する
5. 人間の手作業は認証・デプロイ承認類のみ。CLIで完結できない操作はまとめてチェックリスト形式で提示する
6. デプロイまで Claude Code が実行する
7. **デプロイの確認必須（2026-07-04の事故の恒久対策）**: `main` へ push したら「公開した」と言う前に必ず、①GitHub Actions の該当 run が **success**（`gh run watch <id> --exit-status`）②本番サイトの更新（`curl -s https://yuya591-ux.github.io/hitonatsu/proto3d.html | grep -oE 'assets/proto3d-[A-Za-z0-9_]+\.js'` のハッシュが変わった）——の両方を確認する。CI失敗が続くと購読者に失敗メールが届く。検証系スクリプトのブラウザ実行ファイルは必ず `scripts/browser-path.mjs` の `resolveBrowser` を使う（各所に直書きしない＝環境変数の読み忘れでCIが全滅する）。deployが「try again later」で固着したら、Pages設定を再適用（`gh api --method PUT repos/OWNER/REPO/pages -f build_type=workflow`）→クリーンに1回 dispatch（むやみな連続再実行はメールを増やすので避ける）。

## 差し替え可能性（重要・後からの格上げ前提）
- 背景・キャラクターは「コード描画」で作るが、**後で画像ファイル（Geminiで作った水彩画風の絵など）に差し替えられる構造**にしておく
- 具体的には、描画処理を部品化し、「コードで描く版」と「画像を貼る版」を切り替えられる設計にする。最初はコード描画版で完成させる
- 音声素材も同様に、ファイル差し替えが容易な構造にする

## 著作権（絶対遵守）
- 「僕の夏休み」など既存作品の絵・音・キャラクター・地名・固有の台詞・固有のキャラ名を一切使わない・模倣しない
- 作るのは「あの時代の夏の空気」という普遍的な雰囲気を持つ、完全オリジナルの世界
- 環境音（蝉・風鈴・川・ヒグラシ等）は、ライセンスが明確なフリー素材（CC0等）のみ使用し、出典とライセンスを `CREDITS.md` に記録する

## 情報・素材の誠実性
- 使用したフリー素材の出典・ライセンスURLを必ず `CREDITS.md` に全数記録する
- ライセンス不明の素材は使わない

## 表記
- UI文言・コードコメント・ドキュメントはすべて日本語
- 画面に出す文言は素朴で温かい言葉づかい。システム用語を露出させない
